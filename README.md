# UA-Air-Raid-Dynamics 🇺🇦
> **Time Series Analysis of Air Raids in Ukraine** — високонадійний, відмовостійкий ETL-пайплайн для збору, первинного очищення та детермінованого статистичного аналізу повітряних тривог в Україні.

Проєкт спроєктовано за принципом розподілу відповідальності (**Separation of Concerns**) для виявлення аномальних сплесків військових загроз (масованих ракетно-дронових обстрілів) без використання важких "чорних скриньок" машинного навчання.

---

## 🏗️ Архітектура системи (3-Tier ETL Architecture)

Система повністю ізольована на три незалежні рівні, що дозволяє легко масштабувати або замінювати компоненти (наприклад, підключити реальну СУБД TimescaleDB замість файлових потоків):

1. **Ingestion Layer (`data_loader.py`)**
   * Автономний збір даних із зовнішніх API.
   * Імплементація патерну **Retry** (3 спроби з експоненційним або фіксованим очікуванням) та жорстких таймаутів для захисту від мережевих збоїв (помилки 502/503/504).
   * Професійне логування (`logging`) для інтеграції з ELK/Grafana Loki.
2. **Processing Layer (`analyzer.py`)**
   * Повне очищення "брудних" даних (видалення невалідних структур).
   * **Алгоритм злиття накладань інтервалів (Interval Merging):** коректне об'єднання локальних та глобальних тривог у межах одного регіону без математичного подвоєння часу за добу.
   * **Dynamic Time-Fill:** Динамічне заповнення незавершених тривог (`end_time IS NULL`) поточним часом на момент запиту для розрахунку тривалості в реальному часі.
3. **Presentation Layer (`main.py`)**
   * **Детермінована аналітика:** Розрахунок 7-денного ковзного середнього (Moving Average) та стандартного відхилення (Rolling STD).
   * Автоматичне маркування аномалій через верхню смугу Боллінджера ($\mu + 2\sigma$).
   * Генерація управлінського дашборду для **Decision-Makers** з високим індексом візуальної чистоти (Tufte style).
